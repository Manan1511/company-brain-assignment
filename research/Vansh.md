Component
- Vector Database

Proposed Technology
- Qdrant (Vector Database) integrated with Hugging Face Text Embeddings Inference (TEI) (Embedding Generation Engine).

What does this component do?
- This component functions as the primary Semantic Memory engine, operating in two integrated phases:

Semantic Translation: As the Event Queue routes unstructured engineering data (e.g., incident reports, PRs, Slack discussions) to the Async Workers, the Hugging Face TEI model translates this text into high dimensional numerical vectors. These vectors encapsulate the underlying conceptual meaning of the data.

Storage & Retrieval: Qdrant serves as the storage layer for these embeddings. Upon receiving a query from the Frontend Dashboard, the Search Engine queries Qdrant to perform high-speed Approximate Nearest Neighbor (ANN) calculations. This retrieves highly relevant contextual documents to ground the LLM Layer

Why choose this technology?
- Qdrant and Hugging Face TEI provide an optimal balance of enterprise-grade scalability, deterministic accuracy, and architectural simplicity. By running an open-source Hugging Face model locally, the architecture guarantees strict data sovereignty—ensuring proprietary code and infrastructure secrets never leave internal servers. Qdrant, engineered natively in Rust, offers exceptional memory safety and high-concurrency throughput, operating with a strict computational discipline required for real-time data ingestion Furthermore, Qdrant’s robust Python SDK facilitates the rapid development of simple, aesthetic, and easy-to-use retrieval APIs, bridging the complex backend operations with the frontend systems cleanly.

Advantages
- The system seamlessly merges conceptual search with exact keyword matching (sparse vectors like BM25). Qdrant fuses these dual pipelines natively using Reciprocal Rank Fusion(RRF),delivering the highest precision context to the LLM Layer.
- Qdrant applies payload filters (e.g., routing searches by specific repositories, timestamps, or team ownership) directly within the HNSW graph traversal. This prevents the severe latency bottlenecks typically caused by pre- or post-filtering large vector datasets.
- The custom Rust storage engine effortlessly absorbs the continuous, high-volume write operations generated by the Async Workers without locking the database or degrading concurrent read performance.
- Utilizing localized Hugging Face embedding models ensures that API costs do not scale linearly with data ingestion volume.

Limitations
- Maintaining low-latency search capabilities requires the HNSW index to remain actively loaded in RAM. As the organizational memory graph expands, this component will demand significant and scaling memory provisioning. 
- Operating alongside the Neo4j Graph DB and PostgreSQL Metadata DB introduces complex state management. Robust transactional logic must be implemented within the Async Workers to ensure vectors in Qdrant remain perfectly synchronized with their corresponding relationship nodes in the Graph layer.

Extra
- To counteract Qdrant's high memory consumption, we will implement Scalar Quantization. This compresses the vectors in Qdrant from 32-bit floats to 8-bit integers. This will reduce our Vector DB's RAM usage by up to 75% with almost zero loss in search accuracy, keeping our infrastructure costs down.
- As the Vector DB does not store relationships and this is done by Graph DB. We will enforce a strict UUID (Universally Unique Identifier) policy. The ID of an incident report in Neo4j will be the exact same ID used for the vector payload in Qdrant. This allows the Search Engine to find the semantic meaning in Qdrant, and instantly traverse its blast radius in Neo4j.

Would you use it in Company Brain?
- Definitely. We need our AI to tell the truth by relying on hard facts, not just make stuff up. Qdrant helps us do this by matching exact words and understanding the overall meaning of our search. And because we use a local Hugging Face model, our company's private data never gets leaked to the internet. It’s the safest and most accurate way to feed info to our AI.
